    Class ResourcesController

    Contrôleur responsable de l’exposition des ressources via l’API.

    Remarks

    Il expose les points d’accès REST suivants :

    • GET /resources : liste paginable/filtrable des ressources (recherche texte, catégorie, …).
    • GET /resources/categories : liste des catégories de ressources, avec comptage associé.
    • GET /resources/:slug : récupération des détails d’une ressource individuelle via son slug, avec gestion des ressources premium (via les rôles utilisateur).

    Toute la logique métier et l’accès aux données sont délégués au service ResourcesService.

      • Si la ressource est premium, l’accès n’est autorisé que pour les utilisateurs possédant le rôle premium ou admin.
      • La logique de contrôle d’accès est implémentée dans ResourcesService.findOneBySlugForUser.

      Throws

      NotFoundException si aucune ressource ne correspond au slug.

      Throws

      ForbiddenException si la ressource est premium et que l’utilisateur n’a pas les droits nécessaires.
